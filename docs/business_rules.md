# Бизнес-правила системы велопроката

1. Велосипед не может иметь пересекающиеся активные бронирования.
2. В аренду можно выдавать только велосипед со статусом `available` или `reserved`.
3. После подтверждения брони велосипед переводится в статус `reserved`.
4. После выдачи велосипед переводится в статус `in_rent`.
5. После завершения аренды велосипед возвращается в статус `available`, если не требуется сервис.
6. Стоимость аренды рассчитывается по тарифу, привязанному к велосипеду.
7. Залог фиксируется отдельной платежной операцией.
8. При просрочке рассчитывается штраф по ставке `late_fee_per_hour`.
9. При повреждении оператор может начислить дополнительный штраф.
10. Каждое изменение брони должно быть доступно для последующей синхронизации с 1С.
11. Клиент видит только свои бронирования, оператор — все операционные данные.
12. История платежей и аренд не удаляется, а только закрывается статусами.
