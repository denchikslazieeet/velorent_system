# ВелоРент

Веб-приложение для велопроката на Django: каталог велосипедов, бронирования, панель оператора, клиенты, платежи, аналитика и подготовленная интеграция VK.

## Возможности

- Роли клиента, оператора и администратора.
- Каталог велосипедов с тарифами, статусами, фото и картой точки выдачи.
- Онлайн-бронирование и создание брони оператором по телефону.
- Безопасная установка пароля клиентом по одноразовому коду от оператора.
- Проверка документа клиента перед первой выдачей.
- Выдача, возврат, неявка, штрафы, доплаты и расчет итоговой стоимости.
- История платежей, подтверждение оплаты и квитанция по брони.
- Уведомления на сайте и email-уведомления, если клиент указал почту.
- Подготовленная интеграция VK: вход, привязка профиля и заготовка VK-уведомлений.
- Панель оператора с быстрыми фильтрами, метками внимания и аналитикой.
- Светлая и темная тема интерфейса.
- Демо-наполнение одной командой для показа проекта.

## Быстрый запуск для Windows

Этот способ подходит человеку, который не работает с программированием.

1. Установите Python 3.12 или новее: https://www.python.org/downloads/
2. Во время установки обязательно поставьте галочку **Add python.exe to PATH**.
3. Скачайте проект с GitHub кнопкой **Code -> Download ZIP**.
4. Распакуйте ZIP в обычную папку, например на рабочий стол.
5. Откройте папку проекта.
6. Дважды нажмите файл `start_windows.bat`.
7. Дождитесь запуска. Браузер откроет сайт автоматически.

Адрес сайта:

```text
http://127.0.0.1:8000/
```

Данные для входа оператора:

```text
Логин: operator
Пароль: operator123
```

Данные демо-клиентов:

```text
Логины: customer01 ... customer10
Пароль у всех: Mechabear1001
```

Чтобы остановить сайт, закройте окно запуска или нажмите `Ctrl+C`.

## Что делает start_windows.bat

Файл запуска автоматически:

- создает `.env` из `.env.example`;
- создает виртуальное окружение `.venv`;
- устанавливает зависимости из `requirements.txt`;
- применяет миграции базы данных;
- добавляет демо-клиентов, оператора, тарифы, точку выдачи, велосипеды, брони, аренды и платежи;
- запускает локальный сервер Django.

## Сценарий демонстрации

Этот сценарий удобно пройти на защите или при показе проекта.

1. Войдите оператором: `operator` / `operator123`.
2. Откройте `Панель оператора`.
3. Посмотрите блок `Сегодня важно`: там уже есть новые брони, выдачи, активные аренды и возвраты.
4. В фильтрах бронирований нажмите `Новые`, затем откройте бронь `VR-9001`.
5. Нажмите `Выдать код для пароля`, чтобы показать безопасную установку пароля клиентом.
6. Вернитесь в панель и откройте фильтр `Выдать сегодня`: бронь `VR-9002` показывает, что клиенту нужно проверить документ.
7. Откройте фильтр `Активные`: бронь `VR-9003` можно вести как текущую аренду, а `VR-9004` показывает просроченный возврат.
8. Откройте завершенную бронь `VR-9006`: там есть ожидающая оплата и история платежей.
9. Войдите клиентом, например `customer08` / `Mechabear1001`, и посмотрите карточку ближайшей брони в личном кабинете.
10. Проверьте раздел `Уведомления` и квитанцию по брони.

Команду `python manage.py seed_demo` можно запускать повторно: она обновит презентационные данные и вернет сайт в удобное демо-состояние.

## Ручной запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python manage.py migrate
python manage.py seed_demo
python manage.py runserver
```

## PostgreSQL через Docker

В проекте уже есть `psycopg2-binary`, а `docker-compose.yml` поднимает PostgreSQL и передает Django настройки базы через переменные окружения.

```bash
docker compose up --build
```

После запуска сайт будет доступен на `http://127.0.0.1:8000/`. Данные PostgreSQL хранятся в volume `postgres_data`.

Для ручного перехода без Docker укажите в `.env`:

```env
DB_ENGINE=django.db.backends.postgresql
DB_NAME=velorent
DB_USER=velorent
DB_PASSWORD=velorent
DB_HOST=localhost
DB_PORT=5432
```

Затем выполните:

```bash
python manage.py migrate
python manage.py loaddata backups/sqlite_to_postgres_data.json
```

Если переносить старые данные не нужно, вместо `loaddata` можно выполнить `python manage.py seed_demo`.

## Production-настройки

Для реального размещения поставьте `DEBUG=False`, задайте длинный случайный `SECRET_KEY`, реальные `ALLOWED_HOSTS` и включите HTTPS-настройки:

```env
SECURE_SSL_REDIRECT=True
SESSION_COOKIE_SECURE=True
CSRF_COOKIE_SECURE=True
SECURE_HSTS_SECONDS=31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS=True
SECURE_HSTS_PRELOAD=True
USE_X_FORWARDED_PROTO=True
```

Команда `seed_demo` заблокирована при `DEBUG=False`, если явно не передать `--allow-production`.

Готовая схема размещения на VPS с PostgreSQL, Gunicorn, Caddy и автоматическим
HTTPS описана в [`docs/DEPLOYMENT.md`](docs/DEPLOYMENT.md). Для неё используются
`docker-compose.production.yml`, `Caddyfile` и `.env.production.example`.

Развёртывание на виртуальном хостинге REG.RU Host-0 через Passenger и MySQL,
включая перенос текущих данных PostgreSQL, описано в
[`docs/REG_RU_HOSTING.md`](docs/REG_RU_HOSTING.md).

## 1С-синхронизация

Каждое изменение брони создает `SyncEvent`. Отправить ожидающие события можно командой:

```bash
python manage.py send_onec_sync
```

Для немедленной отправки при создании события включите `ONEC_SYNC_IMMEDIATE=True`. Токен из `ONEC_API_TOKEN` передается во внешний сервис как `Authorization: Bearer ...`.

## Договор аренды и персональные данные

Перед использованием печатного договора укажите в `.env` реальные реквизиты владельца проката и email для обращений по персональным данным:

```env
PERSONAL_DATA_EMAIL=privacy@example.ru
RENTAL_PROVIDER_NAME=ИП Иванов Иван Иванович
RENTAL_PROVIDER_DETAILS=ИНН 000000000000, адрес: г. Чита, ...
```

Если в профиле клиента нет имени или фамилии, система потребует их при бронировании. После подтверждения брони и проверки документа оператор может открыть заполненный договор в карточке брони и распечатать его. Публичный шаблон договора доступен по адресу `/contract-template/`, политика обработки персональных данных — `/privacy/`.

Текст договора и политики является шаблоном для проекта. Перед реальным коммерческим использованием его необходимо проверить с учетом реквизитов владельца проката и применимых требований законодательства.
