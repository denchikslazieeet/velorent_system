# ВелоРент

Веб-приложение для велопроката на Django: каталог велосипедов, бронирования, панель оператора, клиенты, платежи, аналитика и подготовленная интеграция VK.

## Возможности

- Роли клиента, оператора и администратора.
- Каталог велосипедов с тарифами, статусами, фото и картой точки выдачи.
- Онлайн-бронирование и создание брони оператором по телефону.
- Безопасная установка пароля клиентом по одноразовому коду от оператора.
- Проверка документа клиента перед первой выдачей.
- Выдача, возврат, неявка, штрафы, доплаты и расчет итоговой стоимости.
- История платежей, подтверждение оплаты и квитанция по брони.
- Уведомления на сайте и email-уведомления, если клиент указал почту.
- Подготовленная интеграция VK: вход, привязка профиля и заготовка VK-уведомлений.
- Панель оператора с быстрыми фильтрами, метками внимания и аналитикой.
- Светлая и темная тема интерфейса.
- Демо-наполнение одной командой для показа проекта.

## Быстрый запуск для Windows

Этот способ подходит человеку, который не работает с программированием.

1. Установите Python 3.12 или новее: https://www.python.org/downloads/
2. Во время установки обязательно поставьте галочку **Add python.exe to PATH**.
3. Скачайте проект с GitHub кнопкой **Code -> Download ZIP**.
4. Распакуйте ZIP в обычную папку, например на рабочий стол.
5. Откройте папку проекта.
6. Дважды нажмите файл `start_windows.bat`.
7. Дождитесь запуска. Браузер откроет сайт автоматически.

Адрес сайта:

```text
http://127.0.0.1:8000/
```

Данные для входа оператора:

```text
Логин: operator
Пароль: operator123
```

Данные демо-клиентов:

```text
Логины: customer01 ... customer10
Пароль у всех: Mechabear1001
```

Чтобы остановить сайт, закройте окно запуска или нажмите `Ctrl+C`.

## Что делает start_windows.bat

Файл запуска автоматически:

- создает `.env` из `.env.example`;
- создает виртуальное окружение `.venv`;
- устанавливает зависимости из `requirements.txt`;
- применяет миграции базы данных;
- добавляет демо-клиентов, оператора, тарифы, точку выдачи, велосипеды, брони, аренды и платежи;
- запускает локальный сервер Django.

## Сценарий демонстрации

Этот сценарий удобно пройти на защите или при показе проекта.

1. Войдите оператором: `operator` / `operator123`.
2. Откройте `Панель оператора`.
3. Посмотрите блок `Сегодня важно`: там уже есть новые брони, выдачи, активные аренды и возвраты.
4. В фильтрах бронирований нажмите `Новые`, затем откройте бронь `VR-9001`.
5. Нажмите `Выдать код для пароля`, чтобы показать безопасную установку пароля клиентом.
6. Вернитесь в панель и откройте фильтр `Выдать сегодня`: бронь `VR-9002` показывает, что клиенту нужно проверить документ.
7. Откройте фильтр `Активные`: бронь `VR-9003` можно вести как текущую аренду, а `VR-9004` показывает просроченный возврат.
8. Откройте завершенную бронь `VR-9006`: там есть ожидающая оплата и история платежей.
9. Войдите клиентом, например `customer08` / `Mechabear1001`, и посмотрите карточку ближайшей брони в личном кабинете.
10. Проверьте раздел `Уведомления` и квитанцию по брони.

Команду `python manage.py seed_demo` можно запускать повторно: она обновит презентационные данные и вернет сайт в удобное демо-состояние.

## Ручной запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python manage.py migrate
python manage.py seed_demo
python manage.py runserver
```

## Основные страницы

- Главная: `/`
- Каталог: `/catalog/`
- Личный кабинет клиента: `/dashboard/`
- Панель оператора: `/operator/`
- Велосипеды оператора: `/operator/bikes/`
- Клиенты оператора: `/operator/customers/`
- Админка Django: `/admin/`

## Что не загружается в GitHub

В репозиторий не попадают локальные и секретные файлы:

- `.env`
- `db.sqlite3`
- `.venv`
- `cloudflared.exe`
- кеши Python

Для примера настроек используется `.env.example`.
